注意：只关注"我"需要做的事，忽略其他人的任务。

## 识别规则

### 应该提取的（必须是"我"要做的）：
1. **别人给我布置的任务**：如别人对我说"你把报告发给我"、"记得提交XXX"
2. **我主动承诺要做的事**：如我说"明天给你回复"、"我来搞定"
3. **我需要参加的会议/活动**：如"下周三开会"（且我需要参加）
4. **我的 DDL**：如课程作业、项目截止日期等

### 不应该提取的：
- **别人的任务和 DDL**（别人要交的作业、别人的工作安排）
- **我向别人报备/告知的事**（"我现在去开会"、"今晚回家"、"我周五走周二回来"）——这是通知，不是待办
- **已经完成的动作**（"我已经发了"、"搞定了"、已接龙、已回复、已报名）
- 纯粹的闲聊和玩笑
- 模糊到无法执行的内容（"回头再说"、"有空聊"）
- 琐碎的生活小事（"记得带伞"、随口说的"买点水果"）
- 群里与我无关的通知和安排

### 合并规则
同一个人在同一天围绕同一件事的多条消息，应合并为一个任务，而非拆成多个。例如：
- 讨论比赛时间 + 地点 + 人员 → 合并为一个"参加比赛"任务，detail 中包含所有信息
- 多条消息讨论同一个作业的不同细节 → 合并为一个"完成作业"任务

只有当事情确实不同（不同主题、不同时间、不同对象）时才分开。宁可少提，不要重复。

## 输出格式

请严格按以下 JSON 格式输出，不要添加任何其他文字：

```json
{{
  "tasks": [
    {{
      "title": "任务简短描述",
      "detail": "任务详细说明（包含上下文）",
      "deadline": "2025-06-05",
      "deadline_text": "周五之前",
      "time": "18:00-20:00",
      "location": "五四体育场",
      "priority": "high",
      "source_chat": "对话名称",
      "source_message": "原始消息摘要",
      "category": "work"
    }}
  ]
}}
```

字段说明：
- priority: "high" / "medium" / "low"
- category: "work"（工作）/ "life"（生活）/ "social"（社交）/ "other"（其他）
- deadline: ISO格式日期字符串，无法确定时为 null
- time: 具体时间或时间段，无法确定时为 null
- location: 地点，无法确定时为 null
